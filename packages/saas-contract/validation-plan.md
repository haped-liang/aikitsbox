## 二、访谈问卷（通用 + 赛道定制）

### 通用部分（3分钟，建立信任）

```
1. 简单介绍一下您目前在做的业务？
2. 在[合同/记账/菜单]这件事上，您目前是怎么处理的？
   → 自己做？请人做？不管它？
3. 这件事平均每周花您多少时间？或者花多少钱？
4. 如果有一个AI工具帮您做这件事，您最想它帮您解决什么？
```

   ☐ 看不懂条款  ☐ 怕被坑  ☐ 不知道少了什么  ☐ 其他___

7. 如果AI能帮您3分钟审完一份合同并标出风险点，
   您愿意为此付费吗？
   ☐ 愿意，每次 ¥___

## 三、如何找到访谈对象（每个赛道5人）

| 赛道 | 找人的渠道 | 话术 |
|------|-----------|------|
| **A: 小微老板** | 朋友圈/微信群/脉脉 | "在做一个小微企业法律痛点的调研，想请您喝杯咖啡聊10分钟" |
| **B: 个体户** | 小红书私信/豆瓣自由职业小组 | "我也是个体户，在做记账工具的调研，想请教几个问题" |
| **C: 餐饮老板** | 去店里吃饭时聊/美团外卖商家群 | "老板生意好啊！我在做一个帮餐厅写菜单的AI工具，想请教您一个问题" |

> 💡 技巧：不要说是"调研"，说是"请教"。不要一上来就卖产品，先听对方吐槽。

## 四、Go / No Go 决策标准

### 每个赛道的打分（访完5人后打分）

| 评估维度 | 权重 | A:合同审查 | B:AI记账 | C:菜单优化 |
|---------|------|-----------|---------|-----------|
| 痛点强度（受访者自发吐槽的程度） | 30% | — | — | — |
| 付费意愿（≥3/5人说愿意付费） | 30% | — | — | — |
| 使用频率（周/月） | 15% | — | — | — |
| 竞争格局（有没有大厂在做） | 10% | — | — | — |
| 与我们其他方向的协同效应 | 15% | — | — | — |
| **总分** | **100%** | | | |

### 决策规则

```
总分 ≥ 70  +  付费意愿 ≥ 3/5  → 🟢 GO！建MVP
总分 50-70                     → 🟡 再访5人，重新评估
总分 < 50                      → 🔴 放弃此赛道
所有赛道都 < 50                 → 暂停方向D，资源集中到方向A/B/C
```

## 五、如果Go：MVP启动计划

### 不要做完整产品！用"微信群+AI后台"验证

```
Week 1-2: 建立付费意向群
  → 在访谈中说"我们有个内测群，前20人免费"
  → 手动用AI帮他们审合同/记账/写菜单
  → 观察: 他们真的会用吗？用几次？会不会推荐给别人？

Week 3-4: 收钱验证
  → 对群外的人说"首批付费用户¥9.9/月"（象征性收费）
  → 关键指标: 有多少人愿意从免费转为付费？
  → 转化率 > 20% → 真正开始写代码做产品
  → 转化率 < 10% → 需求不够痛，放弃
```

### 只有收了钱才算验证成功！

> "如果你免费提供，用户说'太棒了！'
>  如果你收费¥9.9，用户说'我再想想'，
>  那你就没有找到真正的需求。"
> — Steve Blank《创业者手册》
